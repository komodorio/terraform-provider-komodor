# komodor_klaudia_file (Resource)

Manages a Klaudia knowledge-base or blueprint file.

This resource uploads either a Klaudia knowledge-base file or blueprint file. The file content is read from `source_path` or from the write-only `content` argument and is not persisted in Terraform state.
Terraform stores metadata and a SHA-256 checksum used to detect when the file should be updated.

## Example Usage

```terraform
resource "komodor_klaudia_file" "knowledge_base" {
  type        = "knowledge-base"
  filename    = "platform-runbook.md"
  source_path = "${path.module}/platform-runbook.md"
}

resource "komodor_klaudia_file" "blueprint" {
  type        = "blueprint"
  filename    = "service-blueprint.yaml"
  source_path = "${path.module}/service-blueprint.yaml"
}
```

## Schema

### Required

- `filename` (String) Filename to use in Komodor. Together with `type`, this is used to find an existing remote file.
- `type` (String) Klaudia file type. Must be `knowledge-base` or `blueprint`.

### Optional

- `checksum` (String) SHA-256 checksum used by Terraform to detect file content changes. Computed from `source_path` or `content` when available.
- `clusters` (Block List, Max: 1) Optional cluster include/exclude scope for this file. (see [below for nested schema](#nestedblock--clusters))
- `content` (String, Sensitive) Write-only file content supplied directly in configuration. This value is not persisted in plan or state. Requires Terraform 1.11 or newer.
- `source_path` (String) Local path to read file content from. The path may be stored in state, but the file content is not.

- `size` (Number) Remote file size in bytes.
- `uploaded_at` (String) Timestamp when Komodor last uploaded or updated the file.
